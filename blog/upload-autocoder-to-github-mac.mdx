This guide helps you do one thing:

**Upload a project generated by [AutoCoder](https://www.autocoder.cc/platform?utm_source=blog&utm_medium=latest&utm_campaign=MacGitHubUpload "Build full-stack apps with AutoCoder.cc") from your Mac to GitHub.**

You do not need to learn Git first, and you do not need to understand the code. Just follow the steps below.

---

## Before You Start

You will need:

- A Mac
- A GitHub account
- The ZIP file downloaded from AutoCoder

The ZIP file from AutoCoder usually has a random number as its name, for example:

```
2601877049.zip
```

Double-click the ZIP file to extract it. This creates a folder with the same random number. Open that folder, and you will find an `AutoCoder.cc` folder inside.

The folder structure should look something like this:

```
2601877049.zip
└── Extracted folder: 2601877049
    └── AutoCoder.cc
        ├── app
        ├── public
        ├── src
        ├── package.json
        └── other project files
```

What we actually need to upload is the project content **inside** `AutoCoder.cc`.

In this example, the GitHub repository is called:

```
njz
```

---

## Step 1: Create the `njz` Repository on GitHub

Open [GitHub](https://github.com "Sign in to your GitHub account") and sign in.

1. Click the `+` button in the top-right corner.
2. Click **New repository**.
3. Enter `njz` under **Repository name**.
4. Choose **Public** or **Private**.
5. Click **Create repository**.

In simple terms, you have just created an online folder for your code.

---

## Step 2: Copy the `njz` Repository Link

Open the `njz` repository you just created.

1. Click the green **Code** button.
2. Select **HTTPS**.
3. Click the copy button next to the link.

The copied link should look like this:

```
https://github.com/Ryanbib1/njz.git
```

Keep the GitHub page open. We will return to it later to check the upload.

---

## Step 3: Download `njz` to Your Mac

Create a normal folder on your Mac to store the GitHub project. For example:

```
testnjz
```

Right-click the `testnjz` folder and choose:

```
Open in Terminal
```

If that option appears under **Services**, choose:

```
Services → New Terminal at Folder
```

This works even if it is your first time using Terminal. You do not need to set anything up first.

If you cannot find this option in the right-click menu:

1. Press `Command + Space`.
2. Search for and open **Terminal**.
3. Type `cd` followed by one space.
4. Drag the `testnjz` folder into the Terminal window.
5. Press Return.

It should look something like this:

```bash
cd /your/folder/path/testnjz
```

This also opens the correct folder in Terminal.

Now paste the following command into Terminal:

```bash
git clone https://github.com/Ryanbib1/njz.git
```

Press Return.

If your Mac asks you to install command-line developer tools, click **Install**. When the installation finishes, run the command again.

When you see this message, the repository is being downloaded:

```
Cloning into 'njz'...
```

When it finishes, open `testnjz`. You should now see a new `njz` folder inside.

> You do not need to create the njz folder yourself. The git clone command creates it automatically.

---

## Step 4: Move the AutoCoder Project into `njz`

First, find the random-number ZIP file downloaded from AutoCoder:

1. Double-click the ZIP file to extract it.
2. Open the extracted random-number folder.
3. Open the `AutoCoder.cc` folder inside.
4. You should see project items such as `app`, `src`, `public`, and `package.json`.

Now open two Finder windows:

- The `njz` folder you downloaded from GitHub
- The inside of the `AutoCoder.cc` folder

Select everything **inside** `AutoCoder.cc`, then drag it into `njz`.

The result should look like this:

```
njz
├── app
├── src
├── public
├── package.json
└── other project files
```

Open `njz` and make sure you can see `package.json` directly inside it.

> Do not drag the entire outer AutoCoder.cc folder into njz. That adds an unnecessary extra folder level and may require extra setup when deploying the website. Moving the content inside it is simpler.

Remember: do not move the ZIP file, the random-number folder, or the whole `AutoCoder.cc` folder. Only move the project content inside `AutoCoder.cc`.

If the AutoCoder project contains hidden files whose names begin with `.`, press `Command + Shift + .` in Finder to show them, then move them into `njz` as well. Never delete the `.git` folder that belongs to `njz`.

### If You Already Uploaded the Entire `AutoCoder.cc` Folder

You do not need to create the repository again. Fix the folder structure once:

1. Open `AutoCoder.cc` inside `njz`.
2. Move everything inside it to the outer `njz` folder.
3. Delete the now-empty `AutoCoder.cc` folder.
4. Right-click `njz` and choose **Open in Terminal**.
5. Run:

```bash
git add -A
git commit -m "move project to njz root"
git push
```

Refresh GitHub. If `package.json` appears directly on the repository homepage, the folder structure is now correct.

---

## Step 5: Open Terminal in the `njz` Folder

Return to Finder and locate the `njz` folder.

Right-click `njz` and choose:

```
Open in Terminal
```

Or choose:

```
Services → New Terminal at Folder
```

After Terminal opens, the last line will usually end with:

```
njz %
```

If you see `njz %`, Terminal is in the correct folder.

If the Terminal option does not appear in the right-click menu, open Terminal manually, type `cd` followed by one space, drag the `njz` folder into Terminal, and press Return.

---

## Step 6: Upload the Project

You only need four commands.

### 1. Check Which Files Git Found

```bash
git status
```

If you see project items such as `package.json`, `src/`, and `public/`, continue.

### 2. Select All Files for Upload

```bash
git add .
```

It is normal if this command does not display a message.

### 3. Save This Version

```bash
git commit -m "upload AutoCoder project"
```

Terminal may show a long list of files. Do not worry. Git is simply recording the project files.

### 4. Upload to GitHub

```bash
git push
```

Wait for the command to finish.

If the final output contains:

```
main -> main
```

the upload was successful.

---

## Signing In to GitHub for the First Time

If Terminal says that you are not signed in or do not have permission, run:

```bash
brew install gh
```

When the installation finishes, run:

```bash
gh auth login
```

When asked, choose:

1. `GitHub.com`
2. `HTTPS`
3. `Login with a web browser`

Sign in to GitHub in the browser window that opens and approve access.

Return to the Terminal window for `njz` and run:

```bash
git push
```

---

## Fixing the HTTP2 Error

During this upload, we encountered the following error:

```
Error in the HTTP2 framing layer
```

The fix is simple. Run:

```bash
git config --global http.version HTTP/1.1
```

Press Return, then try the upload again:

```bash
git push
```

This is how the project in this example was successfully uploaded.

---

## Check the Result on GitHub

Return to your browser and refresh the `njz` repository page.

The example repository is:

```
https://github.com/Ryanbib1/njz
```

If you can see `package.json`, `src`, and `public` directly on the repository homepage, you are finished.

## Updating the Project Later

Every AutoCoder download gives you a new, complete project folder.

You can think of an update like this:

```
Replace the old project content with the content from the new AutoCoder.cc folder
Then upload the new version to GitHub
```

Follow these steps:

1. Open your local `njz` folder.
2. Move the old project files to your Desktop as a temporary backup.
3. Do not delete the outer `njz` folder.
4. Extract the new random-number ZIP from AutoCoder.
5. Open the random-number folder, then open `AutoCoder.cc` inside it.
6. Move everything inside `AutoCoder.cc` into `njz`.
7. Right-click `njz` and choose **Open in Terminal**.
8. Run:

```bash
git status
git add -A
git commit -m "update AutoCoder project"
git push
```

After the upload finishes, the project on GitHub will be updated to the new version.

> Replace only the project content inside njz. Do not delete or replace the outer njz folder. Most importantly, never delete its hidden .git folder, because that folder connects your local project to GitHub.

If the old project contains images, `.env` files, data, or other files you added yourself, save them separately first. Delete the temporary backup only after you confirm that the new version works correctly.


## Before You Upload

If the GitHub repository is **Public**, anyone can see the files inside it.

Do not upload passwords, payment information, API keys, or `.env` files.

If `.DS_Store` appears on GitHub, do not panic. It is a system file created automatically by macOS and will not affect the website.

The complete process is:

**Create `njz` → copy its link → download it to your Mac → extract the random-number ZIP → open `AutoCoder.cc` → move its project content into `njz` → open Terminal in `njz` → run the four Git commands.**

Done.
