## Requirement Input

Requirement input is the first step in building an application with AutoCoder. You simply describe the product you want to create using natural language, and the system will automatically understand your needs and generate the corresponding project structure.

### Prompt Optimization

When inputting requirements, AutoCoder helps you optimize your Prompt, making your requirement expression clearer and thereby improving the accuracy of the generated results.

<Frame>
  ![Image](/images/image-1.png)
</Frame>

For example, when you input a simple product idea, the system automatically helps you supplement a more complete description:

**User Original Input:**

> Create a restaurant introduction website

**System Optimized Prompt:**

> Design a restaurant introduction website with the following detailed requirements:
>
> 1. **Functional requirements:** The website should provide an overview of the restaurant, including its history, cuisine features, chef introduction, and contact information. It should also display the menu with images and prices, offer reservation instructions, and include customer testimonials to enhance credibility.
> 2. **Page design:** The main pages include a homepage highlighting the restaurant's unique selling points, an about us page, a menu page with categorized dishes, a chef introduction page, a reservation guide page, a customer testimonials section, and a contact page with location and business hours.

**Benefits of Prompt Optimization:**

- Express product requirements more clearly
- Improve the accuracy of generated content
- Reduce the number of subsequent modifications

### Query Suggestion Auto-Fill

To help you start your project quickly, AutoCoder provides recommended Query examples. The system automatically recommends common application scenarios, such as:

- Restaurant Website
- Personal Portfolio
- SaaS Dashboard
- E-commerce Landing Page

You can directly press `Enter` to select the recommended content, or modify it based on the suggestion.

After submitting your requirements, AutoCoder automatically converts your input into a structured **Requirement List**. This helps you confirm the application structure before generating the Demo and continue to optimize product requirements.

The Requirement List interface mainly consists of two parts:

1. **Display Area**: Shows the application structure parsed by AutoCoder, including pages and functional modules.

## View & Structure Management

### View Switching

On the Demo page, you can switch between different views to intuitively inspect the application's structure and layout across various platforms, including: **Web View**, **App View**, and **Backend View**.

### Controller

The Controller displays the current project's page structure and synchronously updates when the page structure changes (such as adding or deleting pages). Through the Controller, you can quickly locate pages you need to view or edit from a single place, flexibly managing the entire application structure.

## Modification & Iteration

After generating the Demo, AutoCoder provides two ways for you to continue refining the application content.

### 1. Chat Modification

You only need to describe the content you want to adjust using natural language. AutoCoder will automatically update the page structure or component content, enabling rapid iteration without regenerating the entire project.

**Supported Modifications:** Add new pages / Delete pages / Modify page layout / Add new functional modules.

**Example Command:**

> "Add an 'Our Story' page to the website."

  ![Image](/images/image-7.png)

Visual Editing allows you to make visual modifications directly within the generated Demo page. After clicking the `Edit` button to enter edit mode, simply click on a page component to open the corresponding editing panel.

## Version History

During the application building process, AutoCoder automatically records the project's version history. Version management helps you retain important versions during the modification process.

- **Revert to Previous Version**: If a modification result is unsatisfactory, you can quickly revert to a previous state, avoiding repetitive operations and allowing you to confidently try different solutions.
- **Mark/Favorite Version**: When the project reaches a crucial stage, you can manually mark or favorite the current version for easier tracking and management later.
