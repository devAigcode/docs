## Project Launch

Launch the project externally, including Web deployment and domain configuration.

### Web Deployment

**Operation Steps:**

<Steps>
  <Step title="Click Publish">
    Click the "Publish" button in the top right corner of the editor.
  </Step>
  <Step title="Select Web Tab">
    In the pop-up window, select the "Web" tab.
  </Step>
  <Step title="Configure Domain">
    Configure the domain (Subdomain or Custom Domain) as needed, and optionally choose whether to enable "Share to SparkVibe" to share it with the community.
  </Step>
  <Step title="Confirm Publish">
    Click "Publish" to confirm the deployment.
  </Step>
</Steps>

<Frame>
  ![Image](/images/image-23.png)
</Frame>

After the deployment is complete, the system will generate two links: the frontend display page external link, and the **Admin Dashboard** link for managing full-stack data.

<Frame>
  ![Image](/images/image-24.png)
</Frame>

### Subdomain

All users (including the free plan) can obtain a `*.autocoder.cc` Subdomain. In the Subdomain configuration item of the publish pop-up, simply enter your desired prefix to have it automatically assigned.

### Custom Domain

<Note>
  This feature is exclusive to paid plans (the **Standard** plan supports binding 1 domain, and the **Pro** plan supports unlimited binding).
</Note>

**Configuration Process:**

<Steps>
  <Step title="Enter Domain">
    Enter the domain you purchased from a third party in the Custom Domain tab of the publish pop-up.
  </Step>
  <Step title="Add CNAME Record">
    Log in to the console of your third-party domain service provider (e.g., AWS, GoDaddy) and add a CNAME record pointing to the target address provided by AutoCoder.cc.
  </Step>
  <Step title="Verify DNS">
    Return to the AutoCoder.cc interface and click "**Verify**" to check the DNS resolution status.
  </Step>
  <Step title="Upload SSL Certificate">
    Manually upload the SSL certificate files for that domain (`.key` and `.crt/.cer` formats) to enable HTTPS secure access.
  </Step>
</Steps>

## App Launch

Complete App packaging and test distribution.

<Note>
  The APP Launch page will only be displayed when the project requires the creation of an APP
</Note>

### App Store Launch

_Currently, the platform does not support directly pushing and publishing the generated App to Google Play or the Apple App Store. The formal release to app stores requires users to handle compliance and the submission process themselves._

### Test Package Download (Android / iOS)

If you requested the generation of a mobile app in the chat, you can switch to the "App" tab after clicking "Publish".

_Currently, it only supports generating and downloading Android APK test packages for Android phones to scan or download for installation testing. Generating iOS test packages (ipa files) or TestFlight distribution is not supported at the moment._

Get full control of the project.

<Frame>
  ![Image](/images/image-26.png)

### Source Code Export Description

- Exporting the project source code is a core privilege of the paid plans (**Standard** and **Pro**). The free plan does not support this operation.

- After subscribing to a paid plan, you can package and download the complete production-grade code written by AutoCoder.cc (including frontend UI, backend logic, and database schema) to your local machine. After exporting, you can deploy it to your own cloud server or a third-party hosting platform, achieving 100% ownership of your digital assets.
